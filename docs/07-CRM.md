# CRM

Внутренняя CRM предназначена только для сотрудников компании.

Роли CRM:

```text
owner
admin
manager
```

Роль `partner` не даёт доступ к CRM. Арбитражники, вебмастера, прямые партнёры и представители CPA-сетей используют отдельный урезанный кабинет или CPA API и не видят внутренние данные компании.

CRM использует те же JSON-коллекции, что и публичный сайт.

Статусы заявки:

```text
new
assigned
contacted
qualified
selection
offer_sent
negotiation
contract_sent
contract_signed
paid
in_progress
delivered
completed
rejected
duplicate
```

В карточке заявки сотруднику доступны:

- назначение менеджера;
- изменение статуса;
- внутренний комментарий;
- причина отказа или дубля;
- источник, ref, click_id и кампания;
- параметры расчёта и выбранный автомобиль;
- история статусов и назначений.

Именно действия сотрудников в CRM являются источником правдивых статусов для CPA-сетей и партнёрской статистики. Внешним партнёрам передаются только разрешённые обезличенные события.

Контактные и паспортные поля клиента заложены с учётом будущего генератора договоров.
